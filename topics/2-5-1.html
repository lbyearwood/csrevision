2.5.1 Languages questions

What this topic covers: high-level and low-level languages, assembly language, machine code, and the role of translators such as compilers, interpreters and assemblers.

1) State one difference between a high-level language and a low-level language. (1 mark)

Mark scheme

1 mark for a correct point:

  • High-level languages are closer to human language, whereas low-level languages are closer to machine code.
  • High-level languages are usually easier for programmers to read and write.
  • Low-level languages give more direct control over hardware.
  • Accept any other valid point.

2) A programmer is writing a mobile app for a large audience. Explain why a high-level language would usually be suitable. (3 marks)

Mark scheme

1 mark for each correct point:

  • High-level languages are easier to understand and write.
  • They contain built-in functions/libraries that speed up development.
  • They are often portable across different processors or systems.
  • They make it easier to maintain or update a large program.
  • Accept any other valid point.

3) Describe two reasons why a programmer might choose assembly language for an embedded system. (4 marks)

Mark scheme

Award up to 2 marks for each reason:

  • It can give direct control over hardware components/registers.
  • This is useful when the program must interact closely with a device.
  • It can be memory efficient.
  • This is useful because embedded systems may have limited memory.
  • It can produce fast or efficient code for a specific processor.
  • This is useful when performance or timing is important.
  • Accept any other valid point.

4) Identify the translator used to convert assembly language into machine code. (1 mark)

Mark scheme

1 mark for:

  • Assembler.

5) A company is preparing a finished game for release. Describe how a compiler translates the program. (2 marks)

Mark scheme

1 mark for each correct point:

  • A compiler translates the whole source program into machine code before execution.
  • It produces an executable/object file that can be run later.
  • It reports errors after attempting to translate the program.
  • Accept any other valid point.

6) A beginner is testing a short Python program line by line. Describe how an interpreter translates the program. (2 marks)

Mark scheme

1 mark for each correct point:

  • An interpreter translates and executes the program one line or instruction at a time.
  • It stops when it reaches an error.
  • It does not usually produce a separate executable file.
  • Accept any other valid point.

7) A teacher wants students to test small sections of code while learning. Justify why an interpreter may be more suitable than a compiler. (3 marks)

Mark scheme

1 mark for each correct point:

  • An interpreter can run code line by line.
  • This helps students see the effect of each instruction quickly.
  • It can stop at the first error and make debugging easier.
  • It avoids waiting for the whole program to be compiled before testing.
  • Accept any other valid point.

8) Compare compiled and interpreted programs in terms of execution speed and error reporting. (4 marks)

Mark scheme

Award marks for linked comparative points:

  • Compiled programs often run faster because they have already been translated into machine code.
  • Interpreted programs may run more slowly because translation happens during execution.
  • A compiler usually reports errors after translation of the whole program has been attempted.
  • An interpreter usually reports an error when it reaches the line containing the error.
  • Accept any other valid point.

9) A company wants to distribute a finished game without sharing the source code. Explain why compiling the game may help. (3 marks)

Mark scheme

1 mark for each correct point:

  • Compilation creates executable/object code.
  • The user can run the program without being given the source code.
  • The source code is harder for users to copy or modify.
  • The compiled program may run faster for the user.
  • Accept any other valid point.

10) Complete the sentences using the terms compiler, interpreter and assembler. A ______ translates assembly language. A ______ translates the whole source program before execution. An ______ translates and runs code one line at a time. (3 marks)

Mark scheme

1 mark for each correct blank in order:

  • assembler
  • compiler
  • interpreter
