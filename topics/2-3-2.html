2.3.2 Testing & identifying errors questions

What this topic covers: test plans, normal, boundary, invalid and erroneous data, plus identifying syntax, logic and run-time errors.

1) A school is about to use a new attendance program. State why the program should be tested before it is used. (1 mark)

Mark scheme

1 mark for a correct point:

  • To find errors before the program is released or used.
  • To check that the program works as intended.
  • To reduce the chance of incorrect output or crashes.
  • Accept any other valid point.

2) Identify the type of test data for each value when testing an age input that should accept ages from 11 to 18: 15, 11, 18, 4. (4 marks)

Mark scheme

1 mark for each correct identification:

  • 15: normal data.
  • 11: boundary data.
  • 18: boundary data.
  • 4: invalid data.

3) A username must be between 6 and 12 characters long. Complete a test plan by giving one normal value, two boundary values and one erroneous value. (4 marks)

Mark scheme

Award up to 4 marks:

  • One normal value with 7 to 11 characters.
  • One lower boundary value with 6 characters.
  • One upper boundary value with 12 characters.
  • One erroneous value of the wrong data type or unsuitable input, such as symbols only if not allowed.
  • Accept invalid values such as 5 or 13 characters if clearly labelled as invalid rather than erroneous.

4) Explain the difference between invalid data and erroneous data. (3 marks)

Mark scheme

1 mark for each correct point:

  • Invalid data is the correct data type but outside the accepted range or rules.
  • Erroneous data is the wrong data type or not sensible for that input.
  • For example, 25 could be invalid for an age range of 11 to 18, while "eleven" could be erroneous if a number is expected.
  • Accept any other valid point.

5) The code below is intended to output a message if a score is 50 or more. Identify the type of error shown. (1 mark)

IF score >= 50
    OUTPUT "Pass"
ENDIF

Mark scheme

1 mark for:

  • Syntax error.
  • The `THEN` keyword is missing after the condition.

6) A student writes a line of code with a missing bracket. Describe what is meant by a syntax error. (2 marks)

Mark scheme

1 mark for each correct point:

  • An error where the program does not follow the rules of the language.
  • It prevents the program from being translated or run correctly.
  • Examples include missing punctuation, misspelt keywords or incorrect indentation where required.
  • Accept any other valid point.

7) A program calculates the average of three numbers using this expression: average = num1 + num2 + num3 / 3. Identify the type of error and explain the problem. (3 marks)

Mark scheme

Award up to 3 marks:

  • Logic error.
  • The program will run but calculate the wrong result.
  • Only num3 is divided by 3 because brackets are missing.
  • Correct expression: average = (num1 + num2 + num3) / 3.
  • Accept any other valid point.

8) A calculator program works for most inputs but crashes when the user enters 0 as a divisor. Describe what is meant by a run-time error and give one example. (3 marks)

Mark scheme

1 mark for each correct point:

  • A run-time error happens while the program is running.
  • It may cause the program to stop or crash.
  • Example: division by zero, trying to open a missing file, or using a list index that does not exist.
  • Accept any other valid point.

9) A program asks the user for two numbers and divides the first by the second. Explain how testing could identify a possible run-time error. (3 marks)

Mark scheme

1 mark for each correct point:

  • Use test data where the second number is 0.
  • This checks whether the program attempts to divide by zero.
  • The expected result should be an error message or validation, not a crash.
  • The test result can show that extra validation or error handling is needed.
  • Accept any other valid point.

10) Compare syntax, logic and run-time errors. (4 marks)

Mark scheme

Award marks for clear comparative points:

  • A syntax error breaks the rules of the programming language.
  • A logic error allows the program to run but gives an incorrect result or behaviour.
  • A run-time error occurs while the program is running and may cause it to stop.
  • Syntax errors are often found by the translator before execution, whereas logic and run-time errors may require testing to identify.
  • Accept any other valid point.

11) A student tests a password program using only one valid password. Explain why this is not enough testing. (3 marks)

Mark scheme

1 mark for each correct point:

  • It only checks that one normal input works.
  • It does not test boundary cases, such as minimum or maximum password length.
  • It does not test invalid or erroneous inputs.
  • Errors may remain hidden because not enough paths through the program have been tested.
  • Accept any other valid point.
