2.1.1 Abstraction

Practise identifying the important details in a problem and removing detail that is not needed.

1) Define what is meant by abstraction in computational thinking. (2 marks)

Mark scheme

1 mark for each correct point:

  • Abstraction means removing unnecessary detail from a problem.
  • It focuses on the important information needed to solve the problem.
  • It can make a problem easier to understand or model.
  • Accept any other valid point.

2) A program calculates the cost of a cinema ticket. The customer’s favourite film is not used in the calculation. Explain why this detail can be ignored. (3 marks)

Mark scheme

1 mark for each correct point:

  • The favourite film is not relevant to calculating the ticket cost.
  • Ignoring it removes unnecessary detail from the problem.
  • The algorithm only needs details such as age, ticket type or discounts.
  • This makes the problem simpler to solve.
  • Accept any other valid point.

3) Identify three details that may be needed when designing a program to calculate the total cost of an online food order. (3 marks)

Mark scheme

Award 1 mark for each relevant detail:

  • The items ordered.
  • The price of each item.
  • The quantity of each item.
  • Delivery charge.
  • Discount code or offer.
  • Accept any other valid point.

4) Explain one benefit of using abstraction before writing an algorithm. (3 marks)

Mark scheme

1 mark for each correct point:

  • It helps the programmer focus on the important parts of the problem.
  • It reduces complexity by removing irrelevant details.
  • It can make the algorithm easier to design, test or explain.
  • It can reduce the chance of including unnecessary steps.
  • Accept any other valid point.
