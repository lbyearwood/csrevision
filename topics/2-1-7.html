2.1.7 Identifying common errors

Practise spotting syntax, logic and runtime errors in algorithms and programs.

1) A student writes IF score > 50 but forgets the required keyword at the end of the selection. Define what is meant by a syntax error. (2 marks)

Mark scheme

1 mark for each correct point:

  • A syntax error is an error in the rules or grammar of the programming language.
  • It may stop the program from running or being translated.
  • Examples include missing brackets, incorrect keywords or missing punctuation.
  • Accept any other valid point.

2) Explain the difference between a syntax error and a logic error. (4 marks)

Mark scheme

1 mark for each correct point:

  • A syntax error breaks the rules of the programming language.
  • A logic error means the program runs but gives the wrong result or behaviour.
  • Syntax errors are usually detected by the interpreter or compiler.
  • Logic errors may require testing or tracing to find.
  • Accept any other valid point.

3) A program calculates an average by dividing the total by 4, even though there are 5 numbers. Identify the type of error and explain your answer. (3 marks)

Mark scheme

1 mark for each correct point:

  • This is a logic error.
  • The program may still run without crashing.
  • The calculation is wrong because it divides by the wrong number of values.
  • The output average will be incorrect.
  • Accept any other valid point.

4) A total in a shopping basket program is sometimes incorrect. Explain how a trace table can help identify errors in the algorithm. (3 marks)

Mark scheme

1 mark for each correct point:

  • A trace table records the values of variables as an algorithm is followed.
  • It shows how values change step by step.
  • Unexpected values can reveal where the algorithm has gone wrong.
  • It can help find logic errors.
  • Accept any other valid point.
