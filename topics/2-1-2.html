2.1.2 Decomposition

Practise breaking larger problems into smaller parts that are easier to solve.

1) Define what is meant by decomposition. (2 marks)

Mark scheme

1 mark for each correct point:

  • Decomposition means breaking a problem down into smaller sub-problems.
  • Each smaller part can be solved or developed separately.
  • The solutions can be combined to solve the whole problem.
  • Accept any other valid point.

2) A school wants a program to manage homework submissions. Identify four sub-problems that could be produced by decomposition. (4 marks)

Mark scheme

Award 1 mark for each relevant sub-problem:

  • Logging in or identifying the student.
  • Showing the homework tasks.
  • Uploading or submitting work.
  • Recording submission dates and times.
  • Allowing teachers to view submissions.
  • Sending reminders for missing work.
  • Accept any other valid point.

3) Explain how decomposition can help a team develop a large program. (3 marks)

Mark scheme

1 mark for each correct point:

  • Different team members can work on different sub-problems.
  • Each part is easier to design, code and test separately.
  • The completed parts can be combined into the full program.
  • It can make the project easier to manage.
  • Accept any other valid point.

4) A program for a quiz game needs questions, scoring and a final result screen. Describe how this problem could be decomposed. (4 marks)

Mark scheme

1 mark for each correct point:

  • One sub-problem could store or display the questions.
  • One sub-problem could get and check the user’s answers.
  • One sub-problem could update the score.
  • One sub-problem could display the final result.
  • The sub-problems can be designed and tested separately.
  • Accept any other valid point.
