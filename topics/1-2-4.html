1.2.4 Binary units questions

Answer each question, then reveal the hidden mark scheme underneath to check your response.

1) Describe the relationship between bits, nibbles and bytes. (3 marks)

Mark scheme

1 mark for each correct point:

  • A bit is a single binary digit.
  • A bit can be either 0 or 1.
  • A nibble is 4 bits.
  • A byte is 8 bits.
  • Two nibbles make one byte.
  • Accept any other valid point.

2) Complete the missing units in the sequence below. Bit, nibble, byte, kilobyte, __________, __________, terabyte, __________ (3 marks)

Mark scheme

1 mark for each correct answer:

  • Megabyte
  • Gigabyte
  • Petabyte
  • Accept MB, GB and PB if clearly placed in the correct order.

3) Convert 6 megabytes into kilobytes. Show your working. (2 marks)

Mark scheme

  • 1 mark for using 1 MB = 1000 KB.
  • 1 mark for correct answer: 6000 KB.
  • Accept 6144 KB if the student clearly uses 1 MB = 1024 KB.

4) Convert 4 gigabytes into megabytes. Show your working. (2 marks)

Mark scheme

  • 1 mark for using 1 GB = 1000 MB.
  • 1 mark for correct answer: 4000 MB.
  • Accept 4096 MB if the student clearly uses 1 GB = 1024 MB.

5) A school has 250 image files. Each image file is 3 MB. Calculate the total storage needed in megabytes. (2 marks)

Mark scheme

  • 1 mark for multiplying 250 by 3.
  • 1 mark for correct answer: 750 MB.
  • Accept any other valid working leading to the correct answer.

6) A video folder contains 8 files. Each file is 750 MB. Calculate the total storage needed in gigabytes. Show your working. (3 marks)

Mark scheme

  • 1 mark for calculating total size in MB: 8 × 750 = 6000 MB.
  • 1 mark for converting MB to GB by dividing by 1000.
  • 1 mark for correct answer: 6 GB.
  • Accept 5.86 GB if the student clearly uses 1024 MB = 1 GB.

7) An image is 800 pixels wide and 600 pixels high. The colour depth is 24 bits. Calculate the file size in bits. (3 marks)

Mark scheme

  • 1 mark for using image file size = colour depth × image height × image width.
  • 1 mark for substituting the values correctly: 24 × 600 × 800.
  • 1 mark for correct answer: 11,520,000 bits.
  • Accept any other valid working leading to the correct answer.

8) A sound file has a sample rate of 8000 Hz, a duration of 10 seconds and a bit depth of 16 bits. Calculate the file size in bits. (3 marks)

Mark scheme

  • 1 mark for using sound file size = sample rate × duration × bit depth.
  • 1 mark for substituting the values correctly: 8000 × 10 × 16.
  • 1 mark for correct answer: 1,280,000 bits.
  • Accept any other valid working leading to the correct answer.

9) A text file uses 8 bits per character and contains 5000 characters. Calculate the file size in bytes. Show your working. (3 marks)

Mark scheme

  • 1 mark for using text file size = bits per character × number of characters.
  • 1 mark for calculating total bits: 8 × 5000 = 40,000 bits.
  • 1 mark for converting to bytes: 40,000 ÷ 8 = 5000 bytes.
  • Accept any other valid working leading to the correct answer.

10) A student says, “A storage device with 64 GB can always store exactly 64 GB of user files.” Explain why this statement may not be fully accurate. (3 marks)

Mark scheme

1 mark for each correct point:

  • Some storage space may be used by the operating system or system files.
  • Some storage space may be used for metadata or file system information.
  • The advertised capacity may be measured differently from how the computer reports it.
  • Actual usable capacity may be lower than the stated capacity.
  • Storage capacity can be affected by formatting.
  • Accept any other valid point.

11) Explain how changing colour depth, image resolution, sample rate, duration or bit depth can affect storage requirements. (6 marks)

Mark scheme

Use the banded mark scheme first, then the indicative content below to decide the final mark.

Mark range Quality of response Expected content
5-6 Clear, accurate and well-developed response. Applies the answer to the scenario or task and covers most of the relevant points from the indicative content.
3-4 Some accurate explanation with relevant points. Covers several relevant points, but the response may lack detail, balance or clear application.
1-2 Limited response with basic understanding. Includes one or two relevant points, but may be brief, generic or partly inaccurate.
0 No creditworthy response. No relevant answer, or the response is too unclear to award marks.

Indicative content:

  • Increasing colour depth increases the number of bits stored for each pixel.
  • Increasing image resolution means there are more pixels to store.
  • Higher colour depth or resolution increases image file size.
  • Increasing sample rate means more sound samples are stored each second.
  • Increasing duration means the sound is stored for more seconds.
  • Increasing bit depth means more bits are stored for each sound sample.
  • Higher sample rate, duration or bit depth increases sound file size.
  • Reducing these values can reduce storage requirements.
  • A developed answer links the factor to the correct file type and file size impact.
  • Accept any other valid point.
