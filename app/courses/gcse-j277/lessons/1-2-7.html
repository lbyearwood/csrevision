Introduction

Computers store whole numbers using a fixed number of bits. In this lesson, you will add binary integers, identify when an answer does not fit into 8 bits, and use binary shifts to quickly multiply or divide by powers of 2.

Starter question

If an 8-bit box can only hold values from 00000000 to 11111111, what happens when the answer needs a ninth bit?

8-bit register

A ninth bit sits outside the 8-bit storage space.

Part 1 - Binary Addition Rules

Part 1 - Binary Addition Rules

Understand the binary addition rules

Binary addition follows a small set of rules. Each column can only hold 0 or 1, so when a column reaches the value 2, you write 0 and carry 1 into the next column.

CalculationWriteCarryWhy
0 + 00-No value to move.
0 + 11-The column can hold 1.
1 + 01-The column can hold 1.
1 + 101Two is 10 in binary.
1 + 1 + 111Three is 11 in binary.

Part 1 - Binary Addition Rules

Knowledge check questions

Why not 2?

Why does 1 + 1 not equal 2 in binary notation?

Answer

Binary only has the digits 0 and 1. The denary value 2 is written as 10 in binary.

Correct result

What is the correct result of 1 + 1 in binary?

Answer

1 + 1 = 10. You write 0 in the current column and carry 1 to the next column.

Forgotten carry

What would happen if you forgot to carry the 1?

Answer

The answer would be too small because the value that should move left into the next column would be missing.

Part 2 - Adding two 8-bit binary numbers

Knowledge check questions

Start on the right

Why do we start from the rightmost bit to carry out binary addition?

Answer

The rightmost bit is the 1s column. Carries move left into larger place values, so the method must begin on the right.

Carry meaning

What does a carry mean in binary addition?

Answer

A carry is a value moved left into the next column when the current column is too large to store as one bit.

Denary check

How could you use denary to check whether an answer is sensible?

Answer

Convert both binary numbers and the answer to denary, then check that the denary addition matches.

Part 3 - Overflow errors

Exam questions

Q4 mark scheme

A computer adds two 8-bit binary numbers and produces the result 1 01010101. Explain why this causes an overflow error. (2 marks)

Mark scheme

1 mark for identifying that the result needs 9 bits. 1 mark for explaining that only 8 bits can be stored.

Model answer

The result has a ninth bit. An 8-bit register cannot store that extra bit, so overflow has occurred.

Q5 mark scheme

A student adds 11100000 and 01100000. Explain why the answer cannot be fully stored in 8 bits. (2 marks)

Mark scheme

1 mark for saying the full result is too large or needs a ninth bit. 1 mark for linking this to the 8-bit storage limit.

Model answer

11100000 + 01100000 = 1 01000000. The extra leftmost bit cannot fit inside 8 bits.

Q6 mark scheme

A computer stores unsigned 8-bit binary integers. Explain why the denary answer 270 would cause an overflow error. (2 marks)

Mark scheme

1 mark for stating the unsigned 8-bit maximum is 255. 1 mark for explaining that 270 is larger than this limit.

Model answer

Unsigned 8-bit integers can store values from 0 to 255. Since 270 is greater than 255, it cannot be stored in 8 bits.
