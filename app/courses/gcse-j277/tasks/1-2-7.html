Part 1: Binary addition rules

  2. Activity 2: A calculator is adding two binary digits in the 1s column. It gets 1 + 1. Explain why the calculator must write 10 in binary instead of writing 2.

    Binary can only use the digits 0 and 1. The value two is written as 10, so the calculator writes 0 in the current column and carries 1 into the next column.

  3. Activity 3: A phone is adding two tiny binary values: 01 and 01. Draw a two-column example showing where the carry is written.

    Example two-column layout:

    Column2s1s
    Carry1
    Number 101
    Number 201
    Result10

    The 1s column is 1 + 1 = 10, so write 0 in the 1s column and carry 1 into the 2s column.

  4. Challenge: Investigate what 1 + 1 + 1 + 1 is in binary. Explain how you know.

    1 + 1 + 1 + 1 = 100 in binary. Four ones have a denary value of 4, and 4 is written as 100 in binary because it has one 4s column, no 2s column and no 1s column.

Part 3: Overflow errors

Focus: decide whether a result fits inside 8 bits.

  1. Activity 8: State the unsigned 8-bit range.

    The unsigned 8-bit range is 0 to 255.

  2. Activity 9: Decide whether 11110000 + 00110000 causes overflow. Explain your reason.

    Yes. 11110000 is 240 and 00110000 is 48. The total is 288, which is too large for 8 bits. The full binary result is 100100000.

  3. Activity 10: Decide whether 01111111 + 00000001 causes overflow. Explain your reason.

    No. 01111111 is 127 and 00000001 is 1. The result is 10000000, which is 128 and still fits in 8 bits.

  4. Activity 11: Explain why overflow is a storage limit issue.

    The arithmetic may be correct, but the answer cannot be stored if it needs more bits than the storage space provides.

  5. Challenge: Write two different additions that cause overflow in 8 bits.

    Examples: 11111111 + 00000001 and 10000000 + 10000000. Both need a 9th bit.

Part 4: Binary shifts

Focus: carry out left and right shifts and explain the effect on value.

  1. Activity 12: Left shift 00010100 by 1 place. State the effect on the denary value.

    00010100 left shifted by 1 becomes 00101000. The denary value changes from 20 to 40, so it doubles.

  2. Activity 13: Left shift 00000111 by 2 places.

    00000111 left shifted by 2 becomes 00011100. The denary value changes from 7 to 28.

  3. Activity 14: Right shift 00110000 by 2 places. State the effect on the denary value.

    00110000 right shifted by 2 becomes 00001100. The denary value changes from 48 to 12, so it is divided by 4.

  4. Activity 15: Explain what happens to empty spaces after a binary shift.

    Empty spaces are filled with 0. Bits shifted beyond the end are lost.

  5. Activity 16: Explain the mistake: A student moves shifted-out bits back onto the other side.

    That is rotation, not a binary shift for this topic. Shifted-out bits are discarded, and new empty spaces are filled with 0.

  6. Challenge: Write a short rule for predicting the effect of shifting left or right by two places.

    A left shift by two places usually multiplies by 4. A right shift by two places usually divides by 4, with any shifted-out bits lost.
