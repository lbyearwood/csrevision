2.6.1 Writing basic programs questions

Practise writing short code fragments using input, output, variables, operators, selection, loops and simple data access.

1) A school welcome screen asks a student to enter their name. Write code that:

  • inputs the name
  • outputs Hello followed by the name.

You may use OCR Exam Reference Language or Python. (2 marks)

Mark scheme

Award up to 2 marks:

  • Inputs or stores the user's name.
  • Outputs a greeting that includes the stored name.
  • Accept OCR Exam Reference Language or a high-level programming language.
  • Accept any other valid point.

OCR-style pseudocode example:

name = INPUT
OUTPUT "Hello " + name

2) A garden design program calculates the area of a rectangular lawn. Complete code that:

  • calculates the area using length and width variables
  • outputs the area.

You may use OCR Exam Reference Language or Python. (3 marks)

Mark scheme

Award up to 3 marks:

  • Uses suitable variables for length and width.
  • Calculates area using length multiplied by width.
  • Outputs the calculated area.
  • Accept OCR Exam Reference Language or a high-level programming language.
  • Accept any other valid point.

OCR-style pseudocode example:

area = length * width
OUTPUT area

7) A maths program checks whether a number is in the range 10 to 20 inclusive. Write code that:

  • inputs a whole number
  • uses comparison and Boolean operations to test whether it is at least 10 and at most 20
  • outputs In range or Out of range.

You may use OCR Exam Reference Language or Python. (4 marks)

Mark scheme

Award up to 4 marks:

  • Inputs or stores a whole number.
  • Uses suitable comparisons with 10 and 20.

  • Outputs a suitable message for true and false outcomes.

OCR-style pseudocode example:

ELSE
    OUTPUT "Out of range"
ENDIF

11) A password checker rejects short passwords. Write code that:

  • inputs a password

You may use OCR Exam Reference Language or Python. (4 marks)

Mark scheme

Award up to 4 marks:

12) A maths revision app grades a percentage score. Write code that:

  • inputs a percentage score
  • outputs High if the score is 70 or more
  • outputs Pass if the score is 50 or more but less than 70
  • outputs Retry otherwise.

You may use OCR Exam Reference Language or Python. (4 marks)

Mark scheme

Award up to 4 marks:

  • Inputs or stores a percentage score.
  • Uses selection with IF, ELSE IF or ELIF, and ELSE.
  • Tests the 70 and 50 boundaries correctly.
  • Outputs the correct category for each branch.
  • Accept any other valid point.

OCR-style pseudocode example:

score = INPUT
IF score >= 70 THEN
    OUTPUT "High"
ELSE IF score >= 50 THEN
    OUTPUT "Pass"
ELSE
    OUTPUT "Retry"
ENDIF

13) A science experiment repeats readings until a valid pH value is entered. Valid pH values are from 0 to 14 inclusive. Write code that:

  • inputs a pH value
  • uses a while loop to keep asking while the pH value is below 0 or above 14
  • outputs Valid pH after a valid value has been entered.

You may use OCR Exam Reference Language or Python. (4 marks)

Mark scheme

Award up to 4 marks:

  • Inputs or stores a pH value before the loop.
  • Uses a WHILE loop.
  • Uses OR with boundary comparisons for invalid values.
  • Inputs a new value inside the loop and outputs a valid message after the loop.
  • Accept any other valid point.

OCR-style pseudocode example:

pH = INPUT
WHILE pH < 0 OR pH > 14
    pH = INPUT
ENDWHILE
OUTPUT "Valid pH"

20) A science quiz app only lets a student continue if they have entered the correct code word or they are a teacher. The code word is ATOM. Write code that:

  • inputs a code word
  • inputs whether the user is a teacher
  • converts the code word to uppercase before checking it
  • uses a Boolean OR operation to decide whether access is allowed
  • outputs Access granted or Access denied.

You may use OCR Exam Reference Language or Python. (5 marks)

Mark scheme

Award up to 5 marks:

  • Inputs or stores a code word.
  • Inputs or stores teacher status.
  • Uses uppercase conversion before comparing with ATOM.
  • Uses OR to combine the code word check with teacher status.
  • Outputs suitable access granted and denied messages.
  • Accept any other valid point.

OCR-style pseudocode example:

code = UPPER(INPUT)
teacher = LOWER(INPUT)
IF code = "ATOM" OR teacher = "yes" THEN
    OUTPUT "Access granted"
ELSE
    OUTPUT "Access denied"
ENDIF
