2.3.1 Defensive design questions

What this topic covers: designing programs that reduce errors, handle invalid input, use authentication where appropriate and are easier to maintain.

1) State what is meant by defensive design. (1 mark)

Mark scheme

1 mark for a correct point:

  • Designing a program to reduce errors and cope with unexpected or invalid input.
  • Designing a program to prevent misuse and make it more robust.
  • Accept any other valid point.

2) A ticket booking program asks users to enter the number of tickets they want. Identify two validation checks that could be used. (2 marks)

Mark scheme

1 mark for each suitable validation check:

  • Range check, for example between 1 and the maximum number of tickets.
  • Type check, for example the input must be an integer.
  • Presence check, for example the field must not be left blank.
  • Length check where a fixed number of digits is required.
  • Accept any other valid point.

3) A booking form accepts an age of 16 because it is within the allowed range. Explain why validation does not guarantee that this age is accurate. (2 marks)

Mark scheme

1 mark for each correct point:

  • Validation only checks whether data is sensible or follows rules.
  • Incorrect data can still pass validation if it is in an acceptable format or range.
  • For example, a valid date of birth could still be someone else's date of birth.
  • Accept any other valid point.

4) A login system asks for a username and password. Describe two ways authentication can improve the security of the system. (4 marks)

Mark scheme

Award up to 2 marks for each way:

  • It checks the user is who they claim to be.
  • This helps prevent unauthorised users accessing the system.
  • Strong passwords make guessing or brute-force attacks harder.
  • Multi-factor authentication adds another check beyond a password.
  • Account lockout can reduce repeated login attempts.
  • Accept any other valid point.

5) A program asks a user to enter an email address. Complete the missing validation rule: the email address must contain an @ symbol and must not be ______. (1 mark)

Mark scheme

1 mark for:

  • blank / empty / null.
  • Accept equivalent wording.

6) Several programmers maintain a school reports program. Describe how comments can improve the maintainability of the program. (2 marks)

Mark scheme

1 mark for each correct point:

  • Comments explain the purpose of code or complex sections.
  • They help programmers understand the code later.
  • They make it easier to update, debug or fix the program.
  • Accept any other valid point.

7) A program uses variables called x, y and z to store price, discount and final total. Explain why meaningful variable names are part of defensive design. (3 marks)

Mark scheme

1 mark for each correct point:

  • Meaningful names make the purpose of variables clearer.
  • This reduces the chance of a programmer using the wrong variable.
  • It makes the program easier to read, test and maintain.
  • It helps another programmer understand the program more quickly.
  • Accept any other valid point.

8) A program accepts a user's age. It should only accept values from 11 to 18. Write a validation condition that could be used. (2 marks)

Mark scheme

Award up to 2 marks:

  • Checks the lower boundary, for example age >= 11.
  • Checks the upper boundary, for example age <= 18.
  • Example answer: age >= 11 AND age <= 18.
  • Accept any equivalent condition.

9) Compare input validation and authentication. (4 marks)

Mark scheme

Award marks for linked comparative points:

  • Input validation checks whether entered data is acceptable.
  • Authentication checks the identity of a user.
  • Validation helps prevent invalid data being processed.
  • Authentication helps prevent unauthorised access.
  • Both can make a program more robust or secure.
  • Accept any other valid point.

10) A school reports program crashes if a teacher leaves the mark field blank. Explain one defensive design improvement that could reduce this problem. (3 marks)

Mark scheme

1 mark for each correct point:

  • Use a presence check to make sure the mark field is not blank.
  • Display a clear error message asking the teacher to enter a mark.
  • Prevent the program from processing the record until valid data is entered.
  • Use exception handling or safe input handling where appropriate.
  • Accept any other valid point.
