Course overview

OCR GCSE Computer Science J277

The qualification is assessed through two compulsory externally assessed written papers. Both papers are non-calculator papers and all questions are mandatory.

Assessment Structure

Component Paper Marks Duration Weighting Assessment focus
J277/01 Computer systems 80 1 hour 30 minutes 50% Computer systems knowledge and application. The paper includes short, medium and one 8-mark extended response question.
J277/02 Computational thinking, algorithms and programming 80 1 hour 30 minutes 50% Computational thinking, algorithms, programming concepts and practical programming skills.

Paper 2 has Section A worth 50 marks and Section B worth 30 marks. Section B assesses practical programming skills and the ability to design, write, test and refine programs.

Paper 1: Computer Systems

1.1 Systems architecture

  • Architecture of the CPU, including the fetch-execute cycle.
  • CPU components such as the ALU, Control Unit, cache and registers.
  • CPU performance, including clock speed, cache size and number of cores.
  • Embedded systems and examples of their use.

1.2 Memory and storage

  • Primary storage, RAM, ROM, virtual memory and cache.
  • Secondary storage and the suitability of optical, magnetic and solid state storage.
  • Units, binary representation, binary arithmetic, hexadecimal, characters, images, sound and compression.

1.3 Networks, connections and protocols

  • LANs, WANs, network topologies and network hardware.
  • The Internet, DNS, hosting, the cloud and web servers.
  • Wired and wireless networks, performance factors, protocols, layers and encryption.

1.4 Network security

  • Threats including malware, social engineering, brute-force attacks, denial of service, interception and SQL injection.
  • Prevention methods including penetration testing, anti-malware, firewalls, access levels, passwords, encryption and physical security.

1.5 Systems software

  • Operating system functions such as user interface, memory management, multitasking, peripheral management, user management and file management.
  • Utility software, including encryption, defragmentation and data compression tools.

1.6 Impacts of digital technology

  • Ethical, legal, cultural, environmental and privacy impacts.
  • Legislation including data protection, computer misuse, copyright and software licensing.
  • Open source and proprietary software licences.

Paper 2: Computational Thinking, Algorithms And Programming

2.1 Algorithms

  • Computational thinking: abstraction, decomposition and algorithmic thinking.
  • Designing, creating, interpreting, correcting and refining algorithms using pseudocode, flowcharts and programming language.
  • Common errors, trace tables, searching algorithms and sorting algorithms.

2.2 Programming fundamentals

  • Variables, constants, operators, input, output and assignment.
  • Sequence, selection, count-controlled iteration and condition-controlled iteration.
  • Data types, casting, string manipulation, file handling, records, SQL, arrays, sub programs and random number generation.

2.3 Producing robust programs

  • Defensive design, anticipating misuse, authentication and input validation.
  • Maintainability through sub programs, naming conventions, indentation and commenting.
  • Testing, syntax errors, logic errors, test data, test plans and refining algorithms.

2.4 Boolean logic

  • Logic diagrams using AND, OR and NOT.
  • Truth tables and combining Boolean operators to solve problems.

2.5 Languages and IDEs

  • High-level and low-level programming languages.
  • Translators, compilers and interpreters.
  • IDE tools such as editors, error diagnostics, run-time environments and translators.

Practical programming skills

  • Students must have opportunities to design, write, test and refine programs during the course.
  • Paper 2 Section B assesses practical programming skills.
  • Programming answers may use OCR Exam Reference Language or a high-level programming language students know, such as Python.

Assessment Objectives

Objective What students need to do Overall weighting
AO1 Demonstrate knowledge and understanding of computer science. 30%
AO2 Apply knowledge and understanding of computer science. 40%
AO3 Analyse problems, design, program, evaluate and refine solutions. 30%

The specification states that no prior qualification is required. Total guided learning time is 120-140 hours, with total qualification time of 140 hours.
