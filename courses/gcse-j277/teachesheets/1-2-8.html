1.2.8 Characters Lesson

Learn how text characters are represented using binary codes, character sets and logical ordering.

Learning focus

By the end of this lesson, you should be able to explain how characters are represented in binary, describe what a character set is, compare ASCII and Unicode, and explain why the number of bits per character matters.

Key words

  • Character: one letter, number, punctuation mark, space or symbol.
  • Character code: the binary code used to represent one character.
  • Character set: a collection of characters with a unique code for each one.
  • ASCII: a smaller character set often shown using 8 bits in GCSE-style questions.
  • Unicode: a larger character set taught here as using 16 bits per character.

Part 1: What is a character?

A character is one symbol used to write text. It could be a letter, number, punctuation mark, space or special symbol. A computer does not store the visible shape of the character in a text file. It stores a binary code that represents that character.

For example, the capital letter A can be represented by the 8-bit binary code 01000001. The capital letter B has a different code, 01000010. Each character needs its own unique code so the computer knows exactly what to display or process.

Exam Question - Explain unique codes (3 marks)

Explain why each character in a character set must have a unique binary code.

Exam Question answer - Explain unique codes (3 marks)

Each character needs a unique binary code so the computer can tell characters apart. [1]

If two characters had the same code, the computer would not know which one to display or process. [1]

Unique codes allow text to be stored, searched, sorted and displayed accurately. [1]

Part 2: Character sets

A character set is the full collection of characters that a computer can recognise and represent. It links each character to a code. Two important examples are ASCII and Unicode.

ASCII is a smaller character set. It is suitable for basic English letters, numbers and common punctuation. In GCSE-style questions, ASCII is often shown using 8 bits per character.

Unicode is a larger character set. In this GCSE topic, it is taught as using 16 bits per character, which gives 2^16 possible codes. It includes ASCII characters and many more characters from different languages, plus symbols and emojis.

Exam Question - Explain the trade-off (4 marks)

Explain the trade-off between using a smaller character set and a larger character set.

Exam Question answer - Explain the trade-off (4 marks)

A smaller character set can use fewer bits per character, which may reduce storage requirements. [1]

A smaller character set represents fewer characters. [1]

This means it may not support different languages, emojis or specialist symbols. [1]

A larger character set can represent more characters, but Unicode uses 16 bits per character in this topic, so it uses more bits than ASCII. [1]

Part 3: Why bits per character matter

The number of bits used for each character limits how many different codes can be created. More bits means more possible combinations. More combinations means more characters can be represented.

The rule is 2^n, where n is the number of bits. For example, 3 bits gives 2 x 2 x 2 = 8 possible codes. In this topic, Unicode is treated as 16 bits, so it can represent 2^16 = 65,536 possible codes.

Exam Question - Choose the suitable size (4 marks)

A small device needs to store 40 different control symbols. Explain whether 5 bits per character would be enough.

Exam Question answer - Choose the suitable size (4 marks)

5 bits would not be enough. [1]

2^5 gives 32 possible codes. [1]

The device needs 40 different control symbols, so it needs more than 32 codes. [1]

6 bits would be enough because 2^6 gives 64 possible codes. [1]

Part 4: Logical ordering

Character sets are often logically ordered. This means related characters are placed in a sensible sequence. For example, the code for B is one more than the code for A, and the code for C is one more than the code for B.

Logical ordering helps computers compare and sort text. For example, a program can sort names alphabetically because the letter codes follow a predictable order.

Summary

You have learned that computers represent characters using binary codes. A character set links each character to a unique code. ASCII is smaller and is taught here as 8 bits, while Unicode is larger and is taught here as 16 bits. More bits per character allow more possible characters to be represented, and logical ordering helps computers compare and sort text.
