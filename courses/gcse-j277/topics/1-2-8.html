1.2.8 Characters questions

Answer each question, then reveal the hidden mark scheme underneath to check your response.

1) A messaging app stores the text of a message sent by a user. Explain how the characters in the message are represented by a computer. (3 marks)

Mark scheme

1 mark for each correct point:

  • Characters are represented using binary codes.
  • Each character is assigned a unique binary value/code.
  • Each letter, digit or symbol in the message has its own code in the character set.
  • The computer stores and processes the binary code rather than the character itself.
  • A character could be a letter, number, symbol or space.
  • Accept any other valid point.

2) Explain what is meant by a character set. (3 marks)

Mark scheme

1 mark for each correct point:

  • A character set is a collection/list of characters that a computer can represent.
  • Each character in the set is assigned a unique binary code.
  • Character sets allow text to be stored, processed and displayed by a computer.
  • Examples include ASCII and Unicode.
  • Accept any other valid point.

3) Explain the relationship between the number of bits per character and the number of characters that can be represented. (4 marks)

Mark scheme

1 mark for each correct point:

  • More bits per character allow more unique binary codes to be created.
  • More unique binary codes allow more characters to be represented.
  • Fewer bits per character limit the number of characters that can be represented.
  • The number of possible characters is calculated using 2^number of bits.
  • For example, 8 bits can represent 256 different values/characters.
  • Accept any other valid point.

4) Compare ASCII and Unicode. (4 marks)

Mark scheme

1 mark for each correct point:

  • ASCII is a character set used to represent characters in binary.
  • Unicode is a character set used to represent characters in binary.
  • ASCII represents fewer characters than Unicode.
  • Unicode can represent characters from many languages and a wider range of symbols.
  • ASCII is suitable for basic English characters and common symbols.
  • Unicode is more suitable for global communication because it supports many writing systems.
  • Accept any other valid point.

5) A school website needs to display names and messages in several languages. Explain why Unicode is needed even though ASCII already exists. (4 marks)

Mark scheme

1 mark for each correct point:

  • ASCII can only represent a limited number of characters.
  • ASCII is mainly suited to English letters, numbers and common symbols.
  • ASCII may not represent all the characters needed for names and messages in different languages.
  • Unicode can represent a much larger range of characters.
  • Unicode supports characters from different languages.
  • Unicode supports additional symbols, such as emojis or specialist characters.
  • Unicode makes it easier for computers to store and display text from around the world.
  • Accept any other valid point.

6) A student says, “If a character set uses more bits per character, it can represent fewer characters.” Explain why this statement is incorrect. (3 marks)

Mark scheme

1 mark for each correct point:

  • More bits create more possible binary combinations.
  • More binary combinations mean more characters can be represented.
  • The number of possible characters increases as the number of bits increases.
  • For example, 8 bits can represent more characters than 7 bits.
  • Accept any other valid point.

7) Explain why each character in a character set must have a unique binary code. (3 marks)

Mark scheme

1 mark for each correct point:

  • The computer needs to distinguish one character from another.
  • If two characters had the same code, the computer would not know which character to display or process.
  • Unique codes allow text to be stored and retrieved accurately.
  • Unique codes reduce ambiguity when converting between characters and binary.
  • Accept any other valid point.

8) Explain what is meant by character sets being logically ordered. (3 marks)

Mark scheme

1 mark for each correct point:

  • Characters are stored in a sensible sequence/order.
  • Related characters are placed next to each other in the character set.
  • For example, the character code for B is one more than the character code for A.
  • Logical ordering makes it easier to compare, sort or process text.
  • Accept any other valid point.

9) A text file stores 500 characters. Each character uses 8 bits. Calculate the file size in bits. (2 marks)

Mark scheme

  • 500 × 8. [1]
  • 4000 bits. [1]
  • Accept any other valid working.

10) A text file stores 1000 characters. Each character uses 8 bits. Calculate the file size in bytes. (3 marks)

Mark scheme

  • 1000 × 8 = 8000 bits. [1]
  • 8000 ÷ 8. [1]
  • 1000 bytes. [1]
  • Accept any other valid working.
