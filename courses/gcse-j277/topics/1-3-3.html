1.3.3 Client-server vs Peer-to-peer architectures questions

Answer each question, then reveal the hidden mark scheme underneath to check your response.

1) Explain how a client-server network works. (3 marks)

Mark scheme

1 mark for each correct point:

  • Clients request services or resources.
  • A server provides services or resources.
  • The server may provide files, web pages, authentication or storage.
  • The server can manage access to shared resources.
  • Accept any other valid point.

2) Explain how a peer-to-peer network works. (3 marks)

Mark scheme

1 mark for each correct point:

  • Devices can act as both clients and servers.
  • Devices share resources directly with each other.
  • There may be no central server.
  • Each device may manage its own files or resources.
  • Accept any other valid point.

3) Compare client-server and peer-to-peer networks. (4 marks)

Mark scheme

1 mark for each correct point:

  • Client-server networks use a central server to provide services.
  • Peer-to-peer networks allow devices to share resources directly.
  • Client-server networks can be easier to manage centrally.
  • Peer-to-peer networks may be cheaper or simpler for small networks.
  • Client-server networks are more suitable for larger organisations.
  • Peer-to-peer networks may be harder to manage securely as the network grows.
  • Accept any other valid point.

4) Explain one advantage of a client-server network. (2 marks)

Mark scheme

1 mark for each correct point:

  • User accounts or access can be managed centrally.
  • Backups can be managed centrally.
  • Shared files or resources can be stored centrally.
  • Security can be easier to control from the server.
  • Accept any other valid point.

5) Explain one disadvantage of a client-server network. (2 marks)

Mark scheme

1 mark for each correct point:

  • A server can be expensive to buy or maintain.
  • Specialist staff may be needed to manage it.
  • If the server fails, services may become unavailable.
  • More setup and management may be needed.
  • Accept any other valid point.

6) Explain one advantage of a peer-to-peer network. (2 marks)

Mark scheme

1 mark for each correct point:

  • It can be cheaper because no dedicated server is needed.
  • It can be simple to set up for a small number of devices.
  • Devices can share files or resources directly.
  • It may be suitable for a small home network.
  • Accept any other valid point.

7) Explain one disadvantage of a peer-to-peer network. (2 marks)

Mark scheme

1 mark for each correct point:

  • It can be harder to manage security.
  • Files may be stored across different devices.
  • Backups may be harder to manage.
  • It may not work well as the number of devices grows.
  • Accept any other valid point.

8) A school needs central login accounts and shared student files. Recommend a suitable network architecture and justify your answer. (4 marks)

Mark scheme

1 mark for each correct point:

  • Client-server would be suitable.
  • A server can manage user accounts.
  • A server can store shared files centrally.
  • Access rights can be managed centrally.
  • The answer should link the choice to the school scenario.
  • Accept any other valid point.
