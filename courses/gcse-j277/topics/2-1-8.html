2.1.8 Searching algorithms questions

Practise linear search, binary search and tracing search algorithms on simple lists.

1) A library program searches an unsorted list of book IDs for a target ID. Describe how a linear search checks the list. (3 marks)

Mark scheme

1 mark for each correct point:

  • A linear search starts at the first item in the list.
  • It compares each item with the target value.
  • It moves through the list one item at a time until the target is found or the list ends.
  • It can be used on an unsorted list.
  • Accept any other valid point.

2) A list stores the values 8, 3, 12, 5, 9. Trace a linear search for the value 5 and state how many comparisons are made. (4 marks)

Mark scheme

Award up to 4 marks:

  • Compares 8 with 5.
  • Compares 3 with 5.
  • Compares 12 with 5.
  • Compares 5 with 5 and finds the target.
  • States that 4 comparisons are made.
  • Accept any other valid trace.

3) A contacts app uses binary search to find a surname. Explain why the list of surnames must be sorted first. (3 marks)

Mark scheme

1 mark for each correct point:

  • Binary search compares the target with the middle item.
  • It decides which half of the list can be discarded.
  • This decision only works if the values are in sorted order.
  • If the list is not sorted, the target may be incorrectly discarded.
  • Accept any other valid point.

4) A sorted list stores 2, 4, 7, 9, 11, 15, 18. Trace a binary search for the value 15. (4 marks)

Mark scheme

Award up to 4 marks:

  • Checks the middle value 9.
  • Identifies that 15 is greater than 9, so the lower half is discarded.
  • Checks the middle of the remaining half, such as 15.
  • States that 15 is found.
  • Accept any other valid binary search trace.

5) A music app needs to search for a track in a playlist. Compare using linear search and binary search for this task. (4 marks)

Mark scheme

1 mark for each correct point:

  • Linear search checks items one at a time from the start of the list.
  • Binary search repeatedly halves the search area.
  • Linear search can work on unsorted data.
  • Binary search requires sorted data.
  • Binary search is usually faster for large sorted lists.
  • Accept any other valid point.
