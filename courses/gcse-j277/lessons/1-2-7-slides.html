Lesson introduction

Small bits, real calculations

A computer can only work with binary digits, but it still needs to add scores, store values and change numbers quickly.

Think first

How can a computer add numbers using only 0 and 1?

Binary addition uses column addition rules. A carry is used when a column reaches the value two or three.

Learning progression

Build the method in order

This lesson moves from single-column rules to full 8-bit addition, then checks what happens when a result does not fit.

Route

Add bits -> carry -> check denary -> overflow -> shift left and right.

2

3

4

Describe the effect of left and right binary shifts.

Key vocabulary

Five words to use accurately

These terms appear throughout the lesson and in exam questions.

Careful wording

A carry is part of addition. Overflow is what happens when the answer needs more bits than are available.

Binary addition rules

Single-column facts

Binary addition works like denary column addition, but each column can only show 0 or 1.

When the total reaches two, write the binary result and carry into the next column.

Hinge Point Question

Check the carry rule

Use this question to check whether students understand a binary carry before the worked example.

Q11 mark

In binary addition, what should you write for 1 + 1 in one column?

AWrite 2.
BWrite 10 in the same column.
CWrite 0 and carry 1.
DWrite 1 and carry 0.
Before revealing the answer

Ask students to explain why binary cannot write the digit 2 in one column.

1What is 1 + 1 in binary?
2Which bit stays in the current column?
3Which bit is carried to the next column?

Teacher answer panel

Answer hidden

Give students thinking time first. Reveal after discussion.
  1. C

    1 + 1 is 10 in binary, so 0 is written in the current column and 1 is carried.

Reasoning

Binary columns can only show 0 or 1. The value two is represented as 10, so the 1 moves into the next column.

Denary check

Does the binary answer make sense?

Checking in denary helps students spot mistakes in the binary working.

01011 is 11 and 00111 is 7. Their sum is 18, so the binary answer should also represent 18.

Check

Overflow

When the result will not fit

Overflow happens when a calculation result needs more bits than the storage space allows.

With 8 bits, the largest unsigned binary value is 11111111, which is 255 in denary.

Key idea

Overflow is a capacity problem, not a normal carry within the available columns.

Overflow model

One calculation that overflows

11111111 + 00000001 gives 1 00000000 if a ninth bit is allowed.

In an 8-bit space, the ninth bit cannot be stored, so overflow has occurred.

Binary shifts

Left shift increases the value

A left binary shift moves every bit one or more places to the left.

For unsigned positive integers, shifting left by one place has the effect of multiplying by 2, if the result still fits.

Left shift model

Move bits left, fill with zero

Example: 00010110 is 22 in denary.

After one left shift it becomes 00101100, which is 44. The value has doubled.

before00010110
after00101100

22 x 2 = 44

Binary shifts

Right shift decreases the value

A right binary shift moves every bit one or more places to the right.

For unsigned positive integers, shifting right by one place has the effect of dividing by 2 and discarding any bit that falls off.

Right shift model

Move bits right, fill with zero

Example: 00101100 is 44 in denary.

After one right shift it becomes 00010110, which is 22. The value has halved.

before00101100
after00010110

44 / 2 = 22

Hinge Point Question

Check the shift effect

Use this question to check whether students can connect a left shift to its denary effect.

Q21 mark

What is the effect of shifting an unsigned positive binary integer left by one place, assuming no overflow?

AIt divides the value by 2.
BIt doubles the value.
CIt changes the value to denary.
DIt always causes overflow.
Before revealing the answer

Ask students to test a small value, such as 00000100, before choosing.

1Which direction do the bits move?
2What happens to the place values?
3Why does the no-overflow condition matter?

Teacher answer panel

Answer hidden

Give students thinking time first. Reveal after discussion.
  1. B

    A left shift by one place has the effect of multiplying by 2, provided the result fits.

Reasoning

Each bit moves into the next larger place value. If no bit is lost, the value doubles.

Summary recap

Three ideas to remember

Addition

Use column rules and carry when a column reaches 2 or 3.

Overflow

A result overflows when it needs more bits than are available.

Shifts

Left shifts double; right shifts halve for unsigned positive integers.
